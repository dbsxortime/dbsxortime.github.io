###04.05
#구현 프로세스

1. 팀 세팅 유틸 (신규) - 영업실 등 팀 명단 세팅 일괄 처리를 위한 유틸파일
  > 팀리스트
  > 팀 당 직원 리스트
  > 팀 이메일 리스트



2. 민원현황 리포트
  > 종합 조회 SQL작성
    >> 가맹점 별 [당일 개수/가격], [경과 개수/가격]

  > 데이터 가공
    >> 받아온 데이터의 영업실 HashSet [부서 column : 직원이름 > 팀이름 조회 후 Set에 담기 (팀유틸)]
    >> 받아온 데이터의 직원 HashSet [담당자 column]
    >> 받아온 데이터의 가맹점 HashSet [가맹점 column]
    >> 영업실 HashSet을 정렬한 TreeSet (원하는 순서대로 정렬) [부서 column : 팀유틸에서 작성된 팀 순서대로 정렬]

    >> 받아온 데이터의 영업실:직원 HashMap [영업실 : 담당자 매칭]
    >> 받아온 데이터의 직원:가맹점 HashMap [직원 : 가맹점 매칭]
    >> 받아온 데이터의 가맹점:가맹점데이터 HashMap [가맹점 : 데이터 매칭]

    >> 받아온 데이터의 영업실:직원 수 HashMap [영업실 HTML column rowspan]
    >> 받아온 데이터의 직원:가맹점 수 HashMap [직원 HTML column rowspan]

  > HTML구현
    >> 반복 : 영업실 TreeSet
        >> 작성 : 영업실 HTML
        >> 반복 : 직원 HashSet
            >>작성 : 직원 HTML
            >> 반복 : 가맹점 HashSet
                 >> 작성 : 가맹점/데이터 HTML



3. 신규 민원 메일 전송 배치
  > 단위시간 별 신규 민원 체크

  1) 영업실 메일 발송
  > 신규 민원 및 담당자 이메일 조회

  > 데이터 가공
    >> 받아온 데이터의 담당자 이메일 HashSet
    >> 받아온 데이터의 담당자 이름 HashSet

  > HTML구현
    >> 반복 : 이메일 HashSet
        >> 확인 : 담당자 팀 이메일
            >> 반복 : 담당자 HashSet
                >> 조회 : 담당자 민원 List
                >> 작성 : 담당자 민원 HTML
        >> 메일 발송

  2) 가맹점 메일 발송
 > 신규 민원 및 담당자 이메일 조회

  > 데이터 가공
    >> 받아온 데이터의 담당자 이메일 List

  > HTML구현
    >> 반복 : 담당자 List
        >> 조회 : 담당자 민원 List
        >> 반복 : 담당자 민원 List
            >> 작성 : 담당자 신규 민원 HTML
        >> 반복 : 담당자 신규 민원 List
            >> 작성 : 담당자 회신 미경과 민원 HTML
        >> 반복 : 담당자 신규 민원 List
            >> 작성 : 담당자 회신 경과 민원 HTML
        >> 반복 : 민원 담당자 메일 발송
        >> 영업실 메일 발송



4. 신규 가맹점 회신 민원 메일 전송 배치
  > 단위시간 별 신규 가맹점 회신 민원 체크

  > 신규 가맹점 회신 민원 조회

  > HTML구현
    >> 반복 : 신규 가맹점 회신 민원 List
        >> 작성 : 신규 가맹점 회신 민원 HTML
    >> PG가맹점 메일 발송
